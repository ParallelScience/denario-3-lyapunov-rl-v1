The current results reveal a critical trade-off: while the Lyapunov prior (Condition B) improves the *quality* of the final policy (upright stability), it significantly degrades the *reliability* of the learning process. The bimodal performance and high variance in Condition B suggest that the fixed $\Phi(s)$ component acts as a "hard" prior that can trap the agent in suboptimal regions of the value landscape if the initial exploration does not align with the energy-based gradient.

**Critique of Current Methodology:**
1. **The "90% Threshold" Trap:** Your sample efficiency metric is flawed. By defining the threshold relative to each seed's own maximum, you are penalizing Condition B for having a higher ceiling. A better metric would be the absolute return threshold (e.g., "steps to reach -0.5 return"), which would likely show that Condition B reaches functional stability faster than Condition A.
2. **Optimization Conflict:** The "bimodal" failure in Condition B likely stems from the fact that $\Phi(s)$ is a Lyapunov function for the *uncontrolled* system. In SAC, the entropy term and the discount factor $\gamma$ fundamentally change the optimal value function $V^*(s)$ compared to the energy-based $\Phi(s)$. Forcing the critic to learn a residual $f_\theta$ that compensates for both the dynamics *and* the entropy regularization may be creating a non-stationary optimization target.
3. **Initialization Bias:** Initializing the final layer to zero is a good start, but it assumes the scale of $\Phi(s)$ matches the scale of the rewards. If the magnitude of $\Phi(s)$ is significantly larger than the expected discounted returns, the gradients for $f_\theta$ may be suppressed or dominated by the fixed component.

**Actionable Recommendations for Future Iterations:**
1. **Scale the Prior:** Introduce a learnable scalar weight $\alpha$ for the Lyapunov component: $V(s) = \alpha \Phi(s) + f_\theta(s)$. This allows the network to "down-weight" the physical prior if it conflicts with the learned value landscape, potentially reducing the bimodal failure rate.
2. **Shift to Absolute Performance Metrics:** Abandon the "90% of max" metric. Use a fixed, meaningful return threshold (e.g., -0.5 or -1.0) to compare sample efficiency. This will clarify whether Condition B is actually faster at reaching a "good enough" policy.
3. **Analyze the Residual Gradient:** Instead of just plotting the residual, calculate the gradient magnitude of $f_\theta$ versus the gradient magnitude of the fixed $\Phi(s)$ component during the first 10,000 steps. This will confirm if the "bimodal" failure is caused by the prior overwhelming the learning signal.
4. **Simplify the Architecture:** You are currently using SAC (which learns $Q(s,a)$) but your plan mentions PPO (which learns $V(s)$). Ensure you stick to one. If using SAC, the decomposition $Q(s,a) = \Phi(s) + f_\theta(s,a)$ is mathematically sound, but ensure the residual $f_\theta$ is not forced to account for the action-dependence of the Q-function in a way that ignores the state-only nature of $\Phi(s)$.
5. **Test "Soft" Initialization:** Instead of a fixed $\Phi(s)$, try using $\Phi(s)$ as a *regularizer* in the loss function (e.g., $Loss = Loss_{Bellman} + \lambda ||V(s) - \Phi(s)||^2$) rather than a hard architectural constraint. This allows the model to deviate from the prior if the data strongly suggests otherwise, which should stabilize the variance across seeds.
